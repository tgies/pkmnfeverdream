How does it work? We use binjgb, a Game Boy emulator that lets us read and write the emulated console's memory while the game runs.

To know where to write, we rely on pokered, a disassembly of PokÃ©mon Red. A disassembly is source code reconstructed from a compiled binary. It shows the exact memory addresses for things like PokÃ©mon names and sprite data, and how the game expects them to be formatted. We also use it to build a modified ROM that boots straight into battle mode.

The trick is timing. We attach code to the emulator to set a breakpoint. When we see the game about to execute CPU instructions from the address we know is the start of the "load front sprite" code, we write our AI-generated sprite (converted to the Game Boy's tile format) and name to the place where the game expects it, then "skip over" the game's sprite loading code.
